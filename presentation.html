Задача 1. Кластеризация по настроению

Постановка задачи

Гипотеза: пользователи воспринимают музыку не столько по жанрам, сколько по эмоциональному профилю.

Цель: обнаружить типы треков по настроению пользователей во время прослушивания через кластеризацию аудио-характеристик

Почему это важно:

  • Жанровые рекомендации часто промахиваются по настроению пользователя и контексту, в котором он находится
  • Кластеризация по настроению даёт дополнительные признаки для рекомендательной системы \(-\) можно улучшить ее
  • Можно генерировать хорошие контекстуальные плейлисты

Подготовка данных

Выбор признаков - 9 из 13 аудио-характеристик:

danceability, energy, loudness, speechiness, acousticness, instrumentalness, liveness, valence, tempo.

Исключены: key, mode, time_signature - они относятся к теории музыки, а не к эмоциональному восприятию.

Препроцессинг:

  • Стандартизация через StandardScaler
  • Итог: 1 136 690 треков с 9 признаками

EDA: распределения признаков

9 гистограмм показывают, что распределения сильно различаются: acousticness и instrumentalness бимодальны с тяжелыми хвостами по краям распределений, valence и danceability ближе к нормальному. speechless, loadness и liveness асиметричны

EDA: корреляции

Ключевые наблюдения:

  • energy и loudness сильно скоррелированы (0.78) - оба отражают громкость и плотность
  • acousticness отрицательно коррелирует с energy (-0.7) - ожидаемо
  • valence слабо связан со всеми остальными признаками

Временная эволюция

Наблюдения:

  • valence снижается с примерно 0.50 до чуть больше 0.4 за 23 года выборки \(\Rightarrow\) музыка становится грустнее
  • energy стабилен
  • acousticness постепенно растёт
  • danceability тоже постепенно растет

Задача 2. Предсказание популярности треков

Постановка задачи

Цель: предсказать popularity трека (0 - 100) и классифицировать его как хит (popularity >= 70) на основе аудио-характеристик и метаданных.

Главная проблема: сильный дисбаланс классов - хитов всего \(\approx\) 0.3 % от всех треков.

Параметр Значение
Объём тренировки 811 234
Объём теста 348 530
True hits в тесте \(\approx\) 1 039
Доля хитов 0.3 %
Целевой порог popularity \(\geq\) 70

Архитектура

Класс HitPrediction - единый интерфейс над разными имплементациями ML моделей

        ┌──────────────────────────────┐
   X -> │   HitPrediction              │  -> predict_popularity (log-prob)
        │  ─────────────────────────   │  -> predict_hit (0/1)
        │  • деление cat/num           │  -> evaluate (F1, ROC-AUC, CM)
        │  • LabelEncoder для sklearn  │
        │  • native cat features в CB  │
        │  • MODEL_FACTORY: 3 модели   │
        └──────────────────────────────┘
                       │
            ┌──────────┼──────────┐
            ▼          ▼          ▼
         CatBoost   XGBoost   LinearRegression

Конфиги моделей вынесены в task_2/mappings.py.

Конфигурации моделей

CatBoost:

iterations=1000, depth=8, learning_rate=0.03,
loss_function="RMSE", eval_metric="RMSE",
early_stopping_rounds=50

XGBoost:

n_estimators=300, max_depth=6, learning_rate=0.05,
objective="reg:squarederror", n_jobs=-1

LinearRegression - бейзлайн

Бинаризация: predict_hit = (predict_popularity $\geq$ hit_border), по умолчанию hit_border = 70.

Результаты сравнения моделей

Модель F1 (hit) ROC-AUC Время обучения TP FP
LinearRegression 0.0000 0.8251 1.2 c 0 0
CatBoost 0.0038 0.9482 79.8 c 2 5
XGBoost 0.0188 0.9513 13.8 c 10 13

XGBoost выигрывает по ROC-AUC и F1 при пороге 70 и при этом существенно быстрее, чем CatBoost (при обучении и инференсе на CPU)

Confusion matrix

Предсказано: не хит Предсказано: хит
Истина: не хит 346 878 (TN) 13 (FP)
Истина: хит 1 029 (FN) 10 (TP)

Precision = 10 / 23 = 43.5 % - почти половина предсказанных хитов действительно хиты.

Recall = 10 / 1 039 = 0.96 % - но мы ловим меньше 1 % всех хитов.

Выводы по задаче 2

  • XGBoost - рекомендованная имлементация градиентного бустинга: ROC-AUC 0.9513, при этом обучение быстрое
  • F1 низкий не из-за плохой модели, а из-за сильного дисбаланса классов; ROC-AUC как метрика более информативен на таких данных
  • Точки роста:
    • Понижение hit_border до 50 - 60 для увеличения recall
    • class_weight="balanced" или SMOTE/undersampling
    • Добавление temporal-фичей (тренд популярности артиста), social features (упоминания)
    • Двухэтапная стратегия: regress popularity \(\rightarrow\) калибровать порог под бизнес-цель

Бизнес-эксперимент: популярность плейлистов

Гипотеза: ML-курированные плейлисты (treatment) популярнее жанрово-курированных (control).

Сетап:

  • Eligible: 60 жанров с \(\geq\) 200 тестовыми треками
  • Control: 200 плейлистов по 50 треков (по истинным классам)
  • Treatment: 180 плейлистов по 50 треков (по прогнозам)
Метрика Control Treatment \(\Delta\) p-value
Mean popularity 18.73 17.11 -1.62 0.083

Результат - статистически не значим (p > 0.05). Treatment плейлисты чуть менее популярны (но в пределах шума).

Интерпретация: средняя популярность как прокси метрика \(-\) плохой выбор для оценки качества рекомендаций: она сильно зашумлёна и не отражает реальную пользу модели

Бизнес-эксперимент: completion rate

Гипотеза: ML-рекомендации увеличивают долю дослушанных треков.

Симуляция:

Задача 4. Прогнозирование трендов

Постановка задачи

Цель: проанализировать долгосрочные тренды музыки и спрогнозировать их на 5 лет вперёд.

Два направления:

  1. Тренды аудио-характеристик (energy, danceability, valence, acousticness)
  2. Популярность жанров (топ-8 жанров с непрерывной историей)

Подход:

  • Агрегируем датасет по годам \(\rightarrow\) берём медианы признаков (устойчиво к выбросам)
  • Train: \(\leq\) 2018
  • Test: 2019 - 2023
  • Сравниваем 3 модели на одной выборке

Подход: три модели

Модель Что моделирует Параметры
Linear Regression базовый линейный тренд OLS, без сезонности
Exponential Smoothing уровень + тренд (Holt) без сезонной компоненты
SARIMAX (1, 1, 0) AR(1) с дифференцированием p=1, d=1, q=0, без экзогенных

Метрики: MAE и RMSE на тесте 2019 - 2023.

Тренды аудио-признаков

Четыре панели - danceability, energy, valence, acousticness. Тренды 2000 - 2023 + прогноз 2024 - 2028 от трёх моделей.

Направления трендов:

Признак Тренд Интерпретация
energy \(\uparrow\) музыка становится плотнее
danceability \(\uparrow\) больше ритма
loudness \(\uparrow\) громче (loudness war)
tempo \(\uparrow\) темп растёт
acousticness \(\downarrow\) живой звук теряет долю
instrumentalness \(\downarrow\) вокал доминирует
valence \(\approx\) стабилен (с очень медленным снижением)

Топ-8 жанров: динамика популярности

Анализируемые жанры: black-metal, gospel, ambient, acoustic, alt-rock, emo, indian, k-pop.

k-pop показывает взрывной рост в 2010-х; emo и alt-rock теряют долю; gospel и acoustic стабильны.

Прогноз по жанрам

8 панелей (по одной на жанр), на каждой - три модели прогноза до 2028 года.

Прогнозы - направляющие, не абсолютные: SARIMAX и ExpSmoothing близки, LinearRegression сильно расходится на горизонте.

Выводы по задаче 4

  • Музыка движется к более интенсивному, танцевальному, громкому звуку; акустический и инструментальный сегменты сжимаются
  • Holt-Winters / SARIMAX(1,1,0) - оптимальные модели для медианных временных рядов аудио-фичей
  • Линейный тренд плох на горизонте > 2 лет
  • Бизнес-применения:
    • ожидать рост запросов на более энергичные треки, чаще добавлять их в курируемые подборки
    • 5-летние прогнозы - для лучшего планирования инвестиций в авторов в зависимости от жанров

Бонусная задача. Рекомендательная система

Постановка задачи

Цель: для произвольного затравочного трека находить 5 похожих по аудио-характеристикам.

Ограничения:

  • Только аудио - без коллаборативных рекомендаций
  • Должно работать быстро + хорошо масштабироваться на большую музыкальную библиотеку

Архитектура: контентная система на K-Nearest Neighbors.

Подход: KNN на 9 признаках

Параметр Значение

        │
        ▼
   найти seed track в индексе
        │
        ▼
   ball_tree -> 20 ближайших (без самого себя)
        │
        ▼
   similarity = (1 - dist / dist_max) * 100 %
        │
        ▼
   sort & take top-5
